Model metadata is the source of truth for fields, validation, migrations, admin,
serializers, permissions, and ORM query compilation.

## Public packages

| Package | Purpose |
| --- | --- |
| `models` | model metadata, registry, lifecycle state, indexes, constraints, inheritance, permissions, and validation hooks |
| `models/fields` | Django-style field constructors and field-level conversion and validation |
| `models/constraints` | index and constraint metadata helpers |
| `models/hooks` | model lifecycle hooks |
| `models/validation` | structured validation errors |

## Example metadata

```go
package blog

import "github.com/cybersaksham/gogo/models"

func PostMetadata() models.Metadata {
	return models.Metadata{
		AppLabel:  "blog",
		ModelName: "Post",
		TableName: "blog_post",
		Fields: []models.FieldMeta{
			{Name: "id", Column: "id", PrimaryKey: true},
			{Name: "title", Column: "title"},
		},
	}
}
```

Add explicit field names, column names, defaults, null and blank behavior,
choices, validators, indexes, constraints, and custom permissions when those
rules affect database, admin, API, or auth behavior.

Metadata validation rejects duplicate field names or columns, declared fields
without a primary key, duplicate index or constraint names, and duplicate custom
permission codenames. After registering all models, use
`models.Registry.ValidateRelations` to catch unknown relation targets.

Set `Managed` to false for existing tables that Gogo should expose through
ORM/admin/API metadata without including them in migration state.
