## Project checks

```bash
gogo check
gogo check --deploy
go test ./...
```

Generated projects also include a `Makefile` for common workflows.

## CLI commands

| Command | Purpose |
| --- | --- |
| `gogo help` | show available commands |
| `gogo --help` or `gogo -h` | show available commands |
| `gogo version`, `gogo --version`, or `gogo -version` | print CLI version metadata |
| `gogo check` | run settings and system checks |
| `gogo runserver` | run the development HTTP server |
| `gogo startproject` | create a project scaffold |
| `gogo startapp` | create an app scaffold |
| `gogo makemigrations` | create migrations |
| `gogo migrate` | apply migrations |
| `gogo showmigrations` | list migration status |
| `gogo sqlmigrate` | render migration SQL |
| `gogo squashmigrations` | squash migrations |
| `gogo optimizemigration` | optimize migration operations |
| `gogo createsuperuser` | create an admin user |
| `gogo changepassword` | change a user password |
| `gogo collectstatic` | collect static files |
| `gogo shell` | open a project shell |
| `gogo dbshell` | open a database shell |
| `gogo test` | run project tests |
| `gogo worker` | run a queue worker |
| `gogo beat` | run the queue scheduler |
| `gogo inspect` | inspect queue workers |
| `gogo queues` | inspect queues |
| `gogo dumpdata` | dump fixtures |
| `gogo loaddata` | load fixtures |

Report any skipped external integration checks when PostgreSQL, Redis, or
RabbitMQ are not available in local verification.

Generated project `Makefile` commands wrap common workflows:

| Command | Purpose |
| --- | --- |
| `make test` | run project tests |
| `make check` | run `gogo check` |
| `make run` | run the development server |
| `make tidy` | run `go mod tidy` |
