import { Card, CardGrid } from '@astrojs/starlight/components';

Gogo gives Go projects a Django-style backend structure: projects contain apps,
apps own their models and routes, and the framework provides shared packages for
configuration, HTTP, admin, APIs, auth, migrations, ORM, forms, templates,
static files, and background jobs.

<CardGrid>
  <Card title="App-structured projects" icon="setting">
    Generate a project, add apps, and keep models, migrations, admin entries,
    APIs, forms, tasks, permissions, templates, and static files together.
  </Card>
  <Card title="Data layer" icon="document">
    Describe models with metadata, generate migrations, and build SQL through
    the ORM query compiler and dialect packages.
  </Card>
  <Card title="Admin, API, and auth" icon="user">
    Register model admin screens, expose serializers and viewsets, and use the
    built-in user, group, and permission toolkit.
  </Card>
  <Card title="Queues and workers" icon="rocket">
    Register tasks, configure brokers and result backends, run workers and beat,
    and compose workflows with chains, groups, and chords.
  </Card>
</CardGrid>

Start with [installation](./getting-started/installation/) and then create a
project with [the project generator](./getting-started/create-project/).
