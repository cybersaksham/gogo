## Project files

Generated projects separate project-level wiring from app-level feature code:

| Path | Responsibility |
| --- | --- |
| `<project>/settings/` | base, local, test, and production settings constructors |
| `<project>/urls.go` | root route registration |
| `<project>/admin.go` | admin registry and site construction |
| `<project>/middleware.go` | middleware registry for configured middleware names |
| `<project>/lifecycle.go` | server configuration, readiness, and shutdown hooks |
| `<project>/queue.go` | queue app construction |
| `apps/` | first-party app packages |
| `templates/` | project templates |
| `static/` | project static assets |
| `media/` | local uploaded media, ignored by Git |
| `fixtures/` | fixture data |
| `tests/` | project integration tests |
| `deploy/` | Docker and deployment entrypoints |
| `.agent/rules/gogo.md` | base AI-agent rules for client projects |

## App files

Create a new app when a domain owns its own models, routes, APIs, admin entries,
forms, tasks, permissions, or templates.

| File | Responsibility |
| --- | --- |
| `models.go` | model metadata and field definitions |
| `migrations/` | migration files |
| `admin.go` | admin registrations |
| `urls.go` | HTTP routes and handlers |
| `api.go` | API routes and viewsets |
| `serializers.go` | API serializers |
| `forms.go` | HTML forms and model forms |
| `tasks.go` | queue task registration |
| `permissions.go` | permission constants and helpers |
| `services.go` | reusable business logic |
| `templates/<app>/` | app templates |
| `static/<app>/` | app static assets |
| `tests/` | app tests |

Keep shared business rules in services so HTTP, API, admin, tasks, and tests use
the same code path.
