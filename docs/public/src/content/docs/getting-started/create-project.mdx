---
title: Create a Project
description: Generate a Gogo project and add apps to it.
---

## Generate the project

```bash
gogo startproject mysite
cd mysite
cp .env.example .env
```

Generated projects include:

- `go.mod` and `manage.go`
- grouped `.env.example`
- grouped `.gitignore`
- project settings, URLs, middleware, admin, and queue wiring
- `apps/`, `templates/`, `static/`, `media/`, `fixtures/`, and `tests/`
- Docker deployment templates
- `.agent/rules/gogo.md` and `.agent/rules/gogo/*` for AI agents working inside the client project

The generated agent rules include a framework package/feature index, command
list, project layout rules, models/ORM/migrations rules, HTTP/admin/API/auth
rules, forms/templates/static rules, queue rules, settings/security rules, and
testing/deployment rules. Those files are intended to be enough for an AI agent
working inside the client project even when it has no prior Gogo memory.

## Add an app

```bash
go run manage.go startapp blog apps/blog
```

An app keeps its feature code together:

```text
apps/blog/
  app.go
  models.go
  admin.go
  urls.go
  api.go
  serializers.go
  forms.go
  services.go
  tasks.go
  permissions.go
  migrations/
  templates/blog/
  static/blog/
  tests/
```

Creating an app under `apps/<name>` automatically installs it into generated
project settings, `.env.example`, `.env` when present, URLs, admin, queue, and
app config wiring. Then run:

```bash
go mod tidy
go run manage.go check
go run manage.go test ./...
```
