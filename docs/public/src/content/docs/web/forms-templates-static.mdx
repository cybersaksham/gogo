---
title: Forms, Templates, and Static Files
description: Handle HTML forms, templates, static assets, uploads, messages, and i18n.
---

Gogo includes the user-facing pieces needed for traditional server-rendered
workflows.

## Forms

Use forms, widgets, formsets, model forms, field validation, and structured
errors for HTML form workflows. Keep form validation close to user input and
move reusable domain validation into services or model validation helpers.

## Templates

Keep project templates in `templates/` and app templates under
`apps/<app>/templates/<app>/`.

The template package supports rendering, inheritance, tags, filters, context
helpers, safe strings, and template directories from settings.

## Static files and uploads

Keep project static assets in `static/` and app assets under
`apps/<app>/static/<app>/`.

Use static collection and file storage APIs for deployed assets. Keep uploaded
media under `media/` locally and out of Git.

During development, `go run manage.go runserver` serves configured static and
media roots when `GOGO_STATIC_ROOT` and `GOGO_MEDIA_ROOT` are set. For
deployment, run `go run manage.go collectstatic` and serve collected files from
the release artifact, volume, CDN, or fronting web server.

Validate uploads by size, content type, extension, storage path, and
authorization requirements.

## Cross-cutting helpers

Gogo also provides fixtures, cache backends, email backends, request messages,
i18n settings, timezone settings, and signals for cross-cutting notifications.
