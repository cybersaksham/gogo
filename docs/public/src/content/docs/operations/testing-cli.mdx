## Project checks

Inside a generated project, `go run manage.go <command>` is the explicit
Django-style project entrypoint. The installed `gogo` binary also delegates
project-aware commands to `go run manage.go <command>` when it is run from the
project, so settings, routes, admin, app configs, model metadata, fixtures, and
queue task registrations are loaded.

```bash
go run manage.go check
go run manage.go check --deploy
go test ./...
```

Generated apps can contribute `checks.Check` values through `Checks()` and
custom management commands through `Commands()`. Project wiring passes both to
`management.MainProject`, so app checks run through `go run manage.go check`
and app commands run directly, for example `go run manage.go blog.reindex`.

Generated projects also include a `Makefile` for common workflows.

## CLI commands

| Command | Purpose |
| --- | --- |
| `gogo help` | show available commands |
| `gogo --help` or `gogo -h` | show available commands |
| `gogo version`, `gogo --version`, or `gogo -version` | print CLI version metadata |
| `gogo startproject` | create a project scaffold |
| `gogo <project-aware-command>` | delegate to `go run manage.go <project-aware-command>` inside a generated project |
| `go run manage.go check` | run settings and system checks |
| `go run manage.go runserver` | run the development HTTP server with HTTP routes, API routes, admin, and development static/media mounts |
| `go run manage.go startapp` | create and auto-install an app scaffold |
| `go run manage.go makemigrations` | create migrations |
| `go run manage.go migrate` | apply migrations |
| `go run manage.go showmigrations` | list migration status |
| `go run manage.go sqlmigrate` | render migration SQL |
| `go run manage.go squashmigrations` | write a squashed replacement migration |
| `go run manage.go optimizemigration` | optimize migration operations or report no safe changes |
| `go run manage.go createsuperuser` | create an admin user |
| `go run manage.go changepassword` | change a user password |
| `go run manage.go collectstatic` | collect static files |
| `go run manage.go shell` | open a project shell |
| `go run manage.go dbshell` | open a database shell |
| `go run manage.go test` | run project tests |
| `go run manage.go worker` | run a queue worker |
| `go run manage.go beat` | run the queue scheduler |
| `go run manage.go inspect` | inspect queue workers |
| `go run manage.go queues` | inspect queues |
| `go run manage.go dumpdata` | dump fixtures |
| `go run manage.go loaddata` | load fixtures |

Report any skipped external integration checks when PostgreSQL, Redis, or
RabbitMQ are not available in local verification.

Generated project `Makefile` commands wrap common workflows:

| Command | Purpose |
| --- | --- |
| `make test` | run project tests |
| `make check` | run project checks |
| `make run` | run the development server |
| `make tidy` | run `go mod tidy` |
