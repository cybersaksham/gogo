---
title: Settings
description: Configure a Gogo project with environment-backed settings.
---

Gogo settings are loaded into `conf.Settings` from defaults, `.env`, and process
environment values. Generated projects include grouped `.env.example` files.

## Required values

| Key | Purpose |
| --- | --- |
| `GOGO_ENV` | runtime environment: `development`, `test`, or `production` |
| `GOGO_SECRET_KEY` | signing, sessions, CSRF, and tokens |
| `DATABASE_URL` | primary database connection |
| `GOGO_HTTP_ADDR` | web server address |

Production also requires `GOGO_ALLOWED_HOSTS`.

## Common optional values

| Key | Purpose |
| --- | --- |
| `GOGO_DEBUG` | debug mode |
| `GOGO_INSTALLED_APPS` | comma-separated installed app list |
| `GOGO_MIDDLEWARE` | comma-separated middleware list |
| `GOGO_STATIC_URL` and `GOGO_STATIC_ROOT` | static asset serving and collection |
| `GOGO_MEDIA_URL` and `GOGO_MEDIA_ROOT` | uploaded media URLs and storage |
| `GOGO_BROKER_URL` | queue broker |
| `GOGO_RESULT_BACKEND` | task result backend |
| `GOGO_CACHE_URL` | cache backend |
| `GOGO_EMAIL_URL` | email backend |

## Validate settings

Run local checks:

```bash
go run manage.go check
```

Run production checks:

```bash
go run manage.go check --deploy
```

Deploy checks reject unsafe debug settings, weak or missing secrets, missing
allowed hosts, insecure cookies, unreviewed admin paths, unapplied migrations,
uncollected static files, and password reset without email.
