---
title: ORM Queries
description: Build database queries with querysets, expressions, managers, dialects, and transactions.
---

The ORM turns immutable query state into dialect-specific SQL. Querysets return
new querysets instead of mutating the original value.

## Query areas

| Area | Includes |
| --- | --- |
| Connections | database config, database handles, connection registry, health checks, logging |
| Querysets | filters, excludes, ordering, slicing, values, annotations, bulk operations, raw queries |
| Managers | model-specific query entrypoints and typed managers |
| Expressions | `F`, `Value`, `Q`, functions, casts, cases, subqueries, outer refs, exists, windows |
| Lookups | exact, contains, ranges, date parts, null checks, regex, JSON path lookups |
| Aggregates | count, sum, average, min, max, standard deviation, variance |
| Joins | select-related, prefetch-related, join plans, relation metadata |
| Transactions | atomic blocks, nested savepoints, isolation, read-only options |
| Dialects | PostgreSQL and SQLite compilers and feature metadata |

## Compile a query

```go
package docs

import (
	"github.com/cybersaksham/gogo/models"
	"github.com/cybersaksham/gogo/orm"
	"github.com/cybersaksham/gogo/orm/dialects/postgres"
)

func CompilePostQuery() error {
	compiler := orm.NewCompiler(postgres.New())
	query := orm.NewQuery(models.Metadata{
		AppLabel: "blog", ModelName: "Post", TableName: "blog_post",
	}).Select("id", "title")
	_, err := compiler.CompileSelect(query)
	return err
}
```

Use parameterized raw SQL only when the ORM cannot express the query. Keep raw
queries close to the app service that owns the behavior.
