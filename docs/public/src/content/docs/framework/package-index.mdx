Use this page as the public package map for Gogo client projects. Application
code should import these packages from `github.com/cybersaksham/gogo` and must
not import `github.com/cybersaksham/gogo/internal`.

Run `go doc <import-path>` inside a client project for symbol-level details.

## Core project packages

| Import path | Use for |
| --- | --- |
| `github.com/cybersaksham/gogo` | framework entrypoint and versioned public module |
| `github.com/cybersaksham/gogo/app` | app configs, app registry, app discovery resources, app lifecycle, and app management commands |
| `github.com/cybersaksham/gogo/management` | project-local management entrypoint, project commands, checks, settings, routes, models, and queue wiring |
| `github.com/cybersaksham/gogo/conf` | settings defaults, environment loading, validation, and deploy-safe configuration |
| `github.com/cybersaksham/gogo/checks` | system checks, deploy checks, and contrib checks |
| `github.com/cybersaksham/gogo/http` | routers, URL patterns, reversing, request/response helpers, middleware, decorators, redirects, generic views, static mounting, and server startup |
| `github.com/cybersaksham/gogo/security` | CSRF, signing, security headers, and security middleware |
| `github.com/cybersaksham/gogo/sessions` | session data, stores, cookie sessions, and session middleware |
| `github.com/cybersaksham/gogo/health` | health check registration and health responses |

## Agent discovery rule

When an AI agent works in a generated project, it should use this package index
plus the generated `.agent/rules/gogo.md` and `.agent/rules/gogo/*` files as
the framework map. If an exact symbol or option is needed, inspect the public
package with `go doc` or read the installed module under the Go module cache;
do not assume private `internal` package behavior is available to client code.
