---
title: Contrib and Cross-Cutting Features
description: Optional batteries and shared services available to Gogo projects.
---

Gogo includes optional contrib apps and shared service packages beyond the core
models, HTTP, admin, API, auth, and queue features. Install only the pieces a
project uses, keep middleware order explicit, and run system checks after
changing installed apps or service settings.

## Contrib apps

| App | Provides |
| --- | --- |
| `gogo.contrib.sites` | site model, `SITE_ID` style lookup, request host lookup, current-site middleware, admin metadata, migrations, and duplicate-domain checks |
| `gogo.contrib.redirects` | redirect model, 404 redirect middleware, permanent and temporary redirects, gone responses, unsafe target blocking, admin metadata, migrations, and safety checks |
| `gogo.contrib.flatpages` | flat page model, site-filtered page resolution, default and custom templates, registration-required enforcement, admin metadata, and migrations |
| `gogo.contrib.sitemaps` | static and model-backed sitemaps, sitemap index rendering, section rendering, pagination, alternates, priorities, change frequency, last-modified data, and XML escaping |
| `gogo.contrib.syndication` | RSS 2.0 and Atom 1.0 feed rendering with feed metadata, item metadata, object-specific feeds, enclosures, and XML escaping |
| `gogo.contrib.humanize` | template filters such as `apnumber`, `intcomma`, `intword`, `naturalday`, `naturaltime`, and `ordinal` |
| `gogo.contrib.admindocs` | staff-only documentation for models, admin classes, routes, template tags, template filters, settings, and management commands |
| `gogo.contrib.postgres` | PostgreSQL full-text search, trigram helpers, PostgreSQL indexes, validators, and aggregates |
| `gogo.contrib.gis` | PostGIS functions and lookups, geometry wrappers, GDAL-style metadata hooks, layer mapping plans, inspect suggestions, measurements, and geo sitemap helpers |

Site-aware apps should include `gogo.contrib.sites` before redirects,
flatpages, sitemaps, or syndication features that depend on the current site.

## Middleware order

Keep middleware deterministic. Security, host validation, sessions, and auth
should run before middleware that depends on request state. Site middleware
should run before site-aware flatpage and redirect resolution. Redirect
middleware should run late because it inspects 404 responses.

## Shared service packages

| Package | Provides |
| --- | --- |
| `cache` | local, dummy, file, database, Redis, and memcached-style cache APIs with get, set, add, delete, touch, increment, decrement, many-key operations, and cleanup |
| `email` | message construction, alternatives, attachments, MIME rendering, console, dummy, file, memory, and SMTP backends |
| `files` | local storage, safe name normalization, upload objects, memory uploads, temporary uploads, chunked uploads, size limits, path traversal protection, and overwrite avoidance |
| `static` | static file finders, duplicate reporting, hashed filenames, manifest building, and static collection |
| `messages` | request message objects, storage, and middleware |
| `i18n` | language code and timezone helpers |
| `signals` | signal definitions, receiver registration, dispatch, and cross-cutting notifications |
| `health` | health check registration and status reporting |
| `observability` | structured logging, metrics hooks, and tracing hooks |
| `testing` | HTTP client helpers, settings helpers, database helpers, fixture helpers, mail assertions, queue assertions, admin assertions, and framework test support |

## System checks

Run checks after changing installed apps, middleware, service backends, or
deployment settings:

```bash
gogo check
gogo check --deploy
```

Checks should catch missing contrib dependencies, unsupported database dialects,
unsafe redirect settings, missing database extensions, incorrect middleware
order, weak production settings, missing secrets, unapplied migrations, and
uncollected static files.

## Database extensions

`gogo.contrib.postgres` needs the PostgreSQL dialect. Some helpers expect
extensions such as `pg_trgm`, `btree_gin`, `btree_gist`, `hstore`, or
`unaccent`.

`gogo.contrib.gis` needs PostgreSQL with PostGIS enabled:

```sql
CREATE EXTENSION IF NOT EXISTS postgis;
```

Non-PostgreSQL deployments should fail checks before migrations or queries use
PostgreSQL-only or GIS-only features.
