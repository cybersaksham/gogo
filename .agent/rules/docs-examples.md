# Docs And Examples Rules

Use this rule when changing public behavior, generated code, examples, or documentation.

## Documentation

- Update reference docs when public APIs, commands, settings, or generated templates change.
- Update tutorials when user workflows change.
- Update operations docs when deployment, queues, database, admin, security, compatibility, upgrades, dependencies, benchmarks, or release behavior changes.
- Keep README limited to downloading/installing, local setup, contribution, security, and license information.
- Do not put product documentation in README. Do not add feature catalogs, command catalogs, environment variable references, architecture notes, tutorials, API references, implementation status, roadmap details, or product capability descriptions there.
- Put code-maintainer documentation under `docs/code/`, with reference material in `docs/code/reference/`, workflow tutorials in `docs/code/tutorials/`, architecture notes in `docs/code/architecture/`, operational documentation in `docs/code/operations/`, and generated package inventory in `docs/code/generated/`.
- Put public user-facing static documentation under `docs/public/`. Keep it written for framework users and keep it synchronized when public APIs, commands, generated project behavior, or operational requirements change.
- Do not leave stale "planned", "placeholder", or skeleton language in user-facing docs unless describing historical compatibility.

## Generated Docs

- Public package docs are generated by `scripts/verify_docs.go`.
- Run `make docs-generated-update` when public package inventory changes.
- Public static docs are built with Astro Starlight from `docs/public`.
- Keep public docs on the current non-vulnerable Astro/Starlight line. The `docs/public` toolchain requires Node `22.12.0` or newer; Make targets run Astro through `DOCS_NODE` for local verification.
- Run `make docs-public-check` and `make docs-public-build` when changing `docs/public`.

## Examples

- Example apps must compile and test.
- Keep examples using public APIs only.

## Verification

```bash
make docs-verify
make example-blog-test
```
