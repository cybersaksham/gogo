The API package provides request/response wrappers, serializers, parser and
renderer registries, class-style views, viewsets, routers, authentication,
permissions, throttling, filtering, pagination, uploads, metadata, versioning,
and OpenAPI generation.

## Main pieces

| Area | Includes |
| --- | --- |
| Serializers | scalar fields, nested fields, method fields, model serializers, validation errors |
| Views | API views, request hooks, exception handlers, response finalizers |
| Viewsets and routers | model viewsets, action routes, route metadata |
| Auth and permissions | session auth, token auth, permission classes, object permission helpers |
| Filtering | exact field filters, search, ordering, custom filter backends |
| Pagination | page number, limit/offset, cursor pagination |
| Parsers and renderers | JSON, form, multipart, raw bodies, browsable API, plain text |
| OpenAPI | route, serializer, auth, permission, request body, response, and component metadata |

## Serializer example

```go
package docs

import "github.com/cybersaksham/gogo/api"

func ValidateTitle(input map[string]any) bool {
	serializer := api.NewSerializer(api.StringField("title", api.FieldOptions{Required: true}))
	_, _, ok := serializer.Validate(input)
	return ok
}
```

Keep serializers responsible for transport validation and conversion. Put domain
rules in services so HTTP, API, admin, and tasks share behavior.
