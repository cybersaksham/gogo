Gogo ships a built-in auth toolkit. Current client projects should extend or
inherit from the built-in auth models instead of replacing the auth system.

## Models

| Model | Table |
| --- | --- |
| `auth.Permission` | `auth_permission` |
| `auth.Group` | `auth_group` |
| `auth.User` | `auth_user` |

`AbstractBaseUser` and `AbstractUser` provide embeddable metadata for custom
inheritable user types.

## Capabilities

- active users, staff users, superusers, groups, and permissions
- default model permissions: `add`, `change`, `delete`, and `view`
- password hashing with PBKDF2-SHA256 and Argon2ID
- login, password change, password reset, set password, user creation, and user change forms
- session authentication middleware
- decorators for login, permissions, staff, superuser, and custom tests
- password reset token signing and verification

## Authentication example

```go
package docs

import (
	"context"

	"github.com/cybersaksham/gogo/auth"
)

func AuthenticateAdmin() (bool, error) {
	store, err := auth.NewMemoryUserStore(auth.User{
		AbstractUser: auth.AbstractUser{Username: "admin"},
	})
	if err != nil {
		return false, err
	}
	_, ok, err := auth.Authenticate(context.Background(), store, auth.Credentials{Username: "admin"})
	return ok, err
}
```
