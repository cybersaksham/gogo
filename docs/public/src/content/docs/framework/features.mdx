---
title: Feature Map
description: The public feature areas provided by Gogo.
---

Gogo is organized as a single framework made of public Go packages and a CLI.

| Area | What it provides |
| --- | --- |
| Project lifecycle | `gogo startproject`, `gogo startapp`, app registry, settings, middleware, URLs, static folders, templates, fixtures, and deploy templates. |
| Models | model metadata, field metadata, validation hooks, permissions, indexes, constraints, inheritance metadata, content types, and registry APIs. |
| Field types | scalar, numeric, text, boolean, temporal, binary, JSON, file, network, generated, relation, PostgreSQL, and GIS field metadata. |
| ORM | immutable query state, querysets, managers, expressions, lookups, aggregates, functions, joins, prefetch plans, transactions, raw SQL, and SQL compilers. |
| Migrations | operations, autodetection, migration plans, SQL rendering, squashing, optimization, and app migration directories. |
| HTTP | request and response helpers, routing, middleware, decorators, redirects, generic views, URL reversing, and error handling. |
| Admin | model registration, sites, list pages, change forms, delete confirmation, history, filters, search, widgets, inlines, actions, auth views, and permission policies. |
| API | serializers, model serializers, views, viewsets, routers, parsers, renderers, authentication, permissions, pagination, filtering, throttling, uploads, metadata, versioning, and OpenAPI helpers. |
| Auth | users, groups, permissions, content types, password hashing, auth forms, sessions integration, decorators, password reset tokens, and admin metadata. |
| Forms and templates | forms, widgets, formsets, model forms, template rendering, tags, filters, template inheritance, static files, file storage, uploads, messages, and i18n helpers. |
| Queues | task registration, signatures, brokers, result backends, workers, beat schedules, retries, timeouts, routing, rate limits, events, inspection, and canvas workflows. |
| Operations | deploy checks, environment validation, testing helpers, fixture commands, collectstatic, database shell, app shell, health checks, observability, cache, email, and security helpers. |
| Contrib apps | sites, redirects, flatpages, sitemaps, syndication, humanize, admin docs, PostgreSQL helpers, and GIS/PostGIS helpers. |

The framework keeps public application code outside `github.com/cybersaksham/gogo/internal`.
Generated projects should import only public packages such as `app`, `models`,
`orm`, `migrations`, `http`, `admin`, `api`, `auth`, `forms`, `queue`, and
`conf`.

Use the [package index](./package-index/) when an agent or developer needs the
complete public import-path map.
